4 Weighting Algorithms — Comparison & Meta-Bandit Selection
4 algorithms compete · Meta-Bandit (Thompson Sampling) selects best per regime · CPU-only · NumPy
ALGO-1 · BAYESIAN
Bayesian Weighting
87.3
METHOD
Beta-Binomial conjugate prior. Each model has Beta(α, β) posterior. Sample from posterior → weight.
STRENGTH
Uncertainty quantification. Naturally handles small samples. Provides confidence intervals on weights.
WEAKNESS
Slow to adapt to regime changes. Beta prior is binary (win/loss), doesn't use magnitude.
CPU COST
4 Beta samples per cycle = ~2ms (NumPy random)
FORMULA
weight_i = Beta(α_i, β_i).sample() / Σ samples
α_i = α_0 + wins_i · β_i = β_0 + losses_i
ALGO-2 · WEIGHTED VOTING
Exponential Weighted Voting
85.1
METHOD
Exponentially-weighted moving average of per-model Sharpe. Weight ∝ exp(λ × Sharpe_i).
STRENGTH
Simple, fast, interpretable. Single hyperparameter (λ). Smooth weight transitions.
WEAKNESS
No exploration. Can get stuck on one model. Doesn't handle regime changes explicitly.
CPU COST
4 exp() + normalize = ~0.5ms
FORMULA
weight_i = exp(λ × Sharpe_i) / Σ exp(λ × Sharpe_j)
λ = 2.0 (temperature)
ALGO-3 · MAB (THOMPSON)
Multi-Armed Bandit
91.7
METHOD
Thompson Sampling. Each model = arm. Beta posterior per arm. Sample → select → observe → update.
STRENGTH
Optimal exploration-exploitation. Proven regret bound O(√T log T). Naturally explores underperforming models periodically.
WEAKNESS
Stochastic weights (sampled, not deterministic). Can be noisy on short windows.
CPU COST
4 Beta samples + softmax = ~3ms
FORMULA
sample_i ~ Beta(α_i, β_i) · weight_i = softmax(samples, τ)
α_i = 1 + wins_i · β_i = 1 + losses_i · τ = 0.5
ALGO-4 · ONLINE LINEAR
Online Linear Regression
88.9
METHOD
Online gradient descent on linear model: weight_i = w0 + Σ(η × gradient). Learns 7-feature → weight mapping.
STRENGTH
Uses all 7 metrics. Can learn complex interactions. Adapts via SGD with decaying learning rate.
WEAKNESS
Can overfit on noisy short windows. Requires learning rate tuning. Less interpretable.
CPU COST
4×7 matrix-vector + SGD = ~5ms
FORMULA
w ← w - η × ∇L(w) · L = (predicted_weight - optimal_weight)²
η = 0.01 × (1 + epoch)^(-0.5)
META-BANDIT — SELECTS BEST ALGORITHM PER REGIME
Thompson Sampling over 4 algorithms · learns which algorithm produces best weights for each regime
The Meta-Bandit treats each of the 4 weighting algorithms as an "arm." After each cycle, it observes the weight quality (how well the emitted weights performed on the next batch of trades). The Meta-Bandit maintains a Beta(α, β) posterior per algorithm per regime. Per cycle: sample from all 4 algorithm posteriors, select the algorithm with highest sample, use its weights. This means the system automatically switches between Bayesian (good for small samples), MAB (good for exploration), Online Linear (good for complex patterns), and Weighted Voting (good for stable regimes) — without any hardcoded selection logic.
selected_algo(t) = argmax_i { Beta(α_i^regime(t), β_i^regime(t)).sample() }
weight(t) = selected_algo.compute_weights(metrics(t))
Update: α += 1 if weight_quality > threshold, else β += 1
