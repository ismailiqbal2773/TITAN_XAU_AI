Live Intelligent Model Weighting Engine — Architecture
7-stage pipeline · sits under CEO Supervisor · no fixed weights · performance-driven · 4 algorithms compete
CONTEXT — SITS UNDER CEO SUPERVISOR (MODULE 18)
The Weighting Engine receives health scores and control directives from the CEO, computes dynamic weights using 4 competing algorithms, and feeds weights to the Ensemble Voter (Module 11). The CEO can override weights (reduce influence, disable model) — the Weighting Engine respects CEO directives as upper bounds.

CONSTRAINT 1
NO FIXED WEIGHTS
Weights change every cycle (60s). No hardcoded regime→weight mapping.
CONSTRAINT 2
CPU ONLY
NumPy vectorized. No GPU. No PyTorch. Total <30ms per cycle.
CONSTRAINT 3
NO CLOUD
No external API calls. Fully offline. All computation local.
CONSTRAINT 4
NO PAID
No paid services. Open-source only (NumPy, SciPy stats).
CORE PRINCIPLE
Performance-driven allocation — weights emerge from recent performance, not from hardcoded rules
weight_i(t) = f(performance_i(t-1), regime(t), confidence_i(t), CEO_directive_i(t))
∑ weight_i(t) = 1.0 · weight_i(t) ∈ [0, CEO_influence_cap_i]
No fixed mapping: regime → weights. Weights are computed, not looked up.
