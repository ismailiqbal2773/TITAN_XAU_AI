Design Patterns: Strategy (IDetectionRule — 8 interchangeable detectors), Observer (CEOSupervisor subscribes to NATS events), Command (ControlAction — serializable action objects), Facade (CEOSupervisor wraps 5 monitors + 3 engines behind single interface), Repository (Database abstracts persistence), Factory (DetectionEngine creates detectors from config). All classes are fully typed (mypy --strict) — no Any types in public APIs. Zero external LLM dependency — all detection is statistical (NumPy).
