META AI CEO SUPERVISOR LAYER (MODULE 18)
DOES NOT GENERATE TRADING SIGNALS — only monitors, scores, and governs. CEO outputs are influence weights, model enables/disables, and risk mode triggers.
L1 · INGESTION
Multi-Source Subscriber
Subscribes to NATS topics: model predictions, trade outcomes, latency, slippage, broker, regime.
Inputs: XGBoost/LSTM/Transformer/RL preds · fills · MDD · latency · slippage · regime
L2 · ROLLING WINDOWS
4-Window Ring Buffers
Maintains 4 rolling windows per model per metric: 50 / 100 / 250 / 500 trades. Bounded memory.
Windows: 50/100/250/500 trades
Metrics: 8 (WR/PF/Sharpe/Sortino/MDD/Recovery/Latency/Slip)
L3 · HEALTH SCORING
6 Composite Scores
Computes Model Health, Execution Quality, Risk, Broker Quality, Regime Confidence, Overall System Health.
Outputs: ModelHealth · EQS · RiskScore · BQS · RegimeConf · OverallHealth (0-100 each)
L4 · DETECTION
8 Detectors
Runs 8 detectors per cycle: degradation, drift, instability, overfitting, exec deterioration, broker issues, regime failures, latency.
Detectors: 8 · runs every 60s · CPU-optimized · <100ms total
L5 · DECISION
Status Engine
Aggregates 6 scores + 8 detector outputs → GREEN / YELLOW / RED system status.
Output: SystemStatus enum (GREEN/YELLOW/RED) + per-model sub-status
L6 · ACTION
Control Action Engine
Executes 5 control actions: reduce influence, increase influence, disable model, emergency risk reduction, capital preservation.
Actions: 5 · applies to ensemble voter + risk engine + execution engine
L7 · PERSISTENCE
PostgreSQL + Parquet
Time-series health scores → PostgreSQL. Detection events + actions → audit log. Long-term → Parquet.
Tables: 10 (scores + events + actions)
Retention: 7 years (audit) / 90 days (hot)
L8 · REPORTING
Dashboards + Alerts
Prometheus metrics, Grafana dashboard, PagerDuty alerts (RED=P1, YELLOW=P2), Slack #titan-ceo.
Channels: Prometheus · Grafana · PagerDuty · Slack · Audit S3
CONSTRAINT 1
NO PAID APIs
No external paid data sources. All inputs from internal NATS.
CONSTRAINT 2
NO EXT LLM
No OpenAI/Anthropic calls. CPU-only statistical detectors.
CONSTRAINT 3
CPU OPTIMIZED
NumPy vectorized · single-core <100ms per cycle · no GPU required.
CONSTRAINT 4
FULLY OFFLINE
No outbound network required. Optional egress for alerts only.
