Detection Layer — 8 Detectors
8 statistical detectors · run every 60s · CPU-only · fully offline · zero external dependencies
D1 · DEGRADATION
Performance Degradation
Detects declining Sharpe/PF over rolling windows. W50 Sharpe < 0.7 × W250 Sharpe for 3 consecutive cycles.
Inputs: Sharpe W50, W100, W250
Trigger: ratio < 0.7 · 3 cycles
Action: reduce model influence 25%
D2 · DRIFT
Model Drift (PSI)
Detects input feature drift via Population Stability Index. PSI > 0.25 on model input features indicates regime shift.
Inputs: feature distributions W250 vs baseline
Trigger: PSI > 0.25
Action: flag for retraining + reduce influence 15%
D3 · INSTABILITY
Behavioral Instability
Detects consecutive losses or high-variance predictions. 5+ consecutive losses OR prediction std > 2× baseline.
Inputs: trade outcomes, prediction variance
Trigger: 5 consec losses OR σ > 2×
Action: reduce influence 50%
D4 · OVERFITTING
Overfitting Symptoms
Detects IS-OOS gap. Live Sharpe < 0.5 × backtest Sharpe sustained over W250 → strategy overfit to history.
Inputs: live Sharpe vs backtest baseline
Trigger: ratio < 0.5 · W250
Action: disable model (requires manual review)
D5 · EXEC DETERIORATION
Execution Quality Decay
Detects rising latency or slippage. Latency P99 > 1.5× budget OR slippage P90 > 1.5× baseline for 2 consecutive cycles.
Inputs: latency P99, slippage P50/P90
Trigger: 1.5× budget · 2 cycles
Action: flag broker + reduce trade freq
D6 · BROKER ISSUES
Broker-Specific Failures
Detects per-broker anomalies: spread widening, reconnect storms, fill rate drop, position reconciliation failures.
Inputs: per-broker metrics
Trigger: BQS < 70 for 5 cycles
Action: failover to backup broker
D7 · REGIME FAILURES
Regime-Specific Failures
Detects strategy failing in current regime. Per-regime win rate < 40% over W100 in current regime.
Inputs: regime label, per-regime WR
Trigger: regime WR < 40% · W100
Action: suppress entries in that regime
D8 · LATENCY
Latency Budget Breach
Detects signal-to-broker latency breaches. P99 > 150ms budget → stale-signal risk. Triggers risk veto downstream.
Inputs: per-stage latency probes
Trigger: P99 > 150ms · 1 cycle
Action: enable stale-signal veto + alert
DETECTOR ENGINEERING — CPU-OPTIMIZED, FULLY OFFLINE
8 detectors × every 60 seconds = <100ms total per cycle
All 8 detectors use only NumPy vectorized operations — no GPU, no external API calls, no LLM inference. Total CPU time per cycle: <100ms on a single core (measured on 4-vCPU VPS). Memory footprint: ~50 MB for all rolling windows combined. Detectors run as a single asyncio task with 60s sleep between cycles. Fully offline-capable: no outbound network required. Optional egress only for PagerDuty/Slack alerts (which degrade gracefully if network unavailable — alerts queued locally, retried).
