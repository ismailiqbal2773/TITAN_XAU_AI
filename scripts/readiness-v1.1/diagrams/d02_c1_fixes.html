1.5.1
Protocol Buffer Schemas
Cross-language data structures defined in .proto files. C++ and Python generate code from same schemas — boundary contract cannot drift.
Schemas: Tick, Signal, Order, Fill, Position, RiskDecision. All have timestamp, source_id, correlation_id fields for tracing.
1.5.2
Ownership Semantics
C++ owns tick data (heap-allocated, RAII-managed). Python borrows via zero-copy numpy arrays (PyO3 memoryview). No data duplication across boundary.
Pattern: C++ exposes mmap'd tick buffer → Python reads via numpy zero-copy. 87-feature vector computed in C++, exposed to Python as np.ndarray without copy.
1.5.3
Error Propagation
Python exceptions → C++ py::error_already_set → caught at boundary → converted to Status::INTERNAL with error message.
Pattern: All Python calls wrapped in try/except at C++ boundary. Errors logged with correlation_id, propagated as gRPC status codes. No silent failures.
1.5.4
GIL Management
Python GIL released during C++ compute-intensive operations (tick processing, feature engineering). Re-acquired only for Python callbacks.
Pattern: py::gil_scoped_release in C++ tight loops. GIL held <5% of wall-clock time. Enables true parallelism between C++ execution core and Python AI layer.
1.5.5
Build Integration
CMake builds C++ core + PyO3 bindings as single .so. Python imports via import titan_core. CI verifies both C++ tests (GoogleTest) and Python tests (pytest) pass against same binary.
