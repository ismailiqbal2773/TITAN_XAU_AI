ANCHORED WFA
Train window expands with each fold
PHASE 1 · TRAIN
Initial 6-month train window
Strategy parameters optimized on the first 6 months of data. Train window GROWS by 1 month each fold (anchored at start of dataset).
PHASE 2 · VALIDATE
1-month out-of-sample validation
Optimized parameters tested on the next 1 month (not seen during training). Used for parameter selection / early stopping.
PHASE 3 · TEST
1-month final out-of-sample test
Locked parameters tested on the subsequent 1 month. The test result is the official out-of-sample P&L — never used for optimization.
PHASE 4 · ROLL FORWARD
Expand train window, advance all windows by 1 month
Train window now spans months 1-7, validate month 8, test month 9. Repeat until end of dataset. Produces 5-7 folds for a 12-month dataset.
ROLLING WFA
Train window slides forward (fixed length)
PHASE 1 · TRAIN
Fixed 6-month rolling train window
Train window SLIDES forward 1 month each fold (drops oldest month, adds newest). Better for non-stationary markets — old data is discarded.
PHASE 2 · VALIDATE
1-month out-of-sample validation
Identical to anchored — used for parameter selection and early stopping. Same window length (1 month) for consistency.
PHASE 3 · TEST
1-month final out-of-sample test
Identical to anchored — locked parameters tested on subsequent month. The OOS test result.
PHASE 4 · ROLL FORWARD
Slide train window forward by 1 month
Train window drops month 1, adds month 7. Validate becomes month 8, test month 9. Produces 7 folds for a 12-month dataset (more folds than anchored).

OOS COVERAGE
≥ 33%
Out-of-sample test months / total months. Floor for valid WFA.
WFE TARGET
≥ 50%
