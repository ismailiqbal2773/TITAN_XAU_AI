Train / Validate / Test / Roll-Forward — Four Phases
Strict data separation · No parameter leak · Each phase has explicit inputs, outputs, and prohibited actions
1 · TRAINIS
INPUT
6 months of in-sample tick data (broker tick stream)
PURPOSE
Parameter optimization via grid / Bayesian search
METHOD
Walk-forward grid search · 1000-5000 param combos
OBJECTIVE
Maximize IS Sharpe with MDD < 10% constraint
OUTPUT
Top-10 parameter sets ranked by IS Sharpe
PROHIBITED
Any access to validate / test data
2 · VALIDATEOOS-1
INPUT
Next 1 month of out-of-sample tick data
PURPOSE
Parameter selection from top-10 IS candidates
METHOD
Run each top-10 candidate on validate month
OBJECTIVE
Select the candidate with best validate Sharpe
OUTPUT
Single locked parameter set per fold
PROHIBITED
Re-optimization on validate data
3 · TESTOOS-2
INPUT
Next 1 month of out-of-sample tick data (final)
PURPOSE
Official OOS performance — never used for any decision
METHOD
Run locked parameters from validate phase
OBJECTIVE
Record test P&L, Sharpe, MDD for this fold
OUTPUT
Fold test result (added to OOS ledger)
PROHIBITED
Any parameter adjustment based on test result
4 · ROLL FORWARD→
INPUT
End-of-fold state + 1 month of new data
PURPOSE
Advance all three windows by 1 month
METHOD
Anchored: expand train. Rolling: slide train (drop oldest)
OBJECTIVE
Move to next fold · preserve previous test results
OUTPUT
Next fold ready for training · previous OOS locked
PROHIBITED
Modifying prior fold test results
