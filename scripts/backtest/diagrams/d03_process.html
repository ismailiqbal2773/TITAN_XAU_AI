Backtesting Process — 6 Stages
From raw tick data to certification-ready strategy · End-to-end runtime ~12 minutes per 12-month backtest on 4-core VPS
STAGE 01
Data Acquisition
Acquire tick data from broker (or Dukascopy for cross-validation), economic calendar, broker swap schedule, dividend schedule (none for XAU but format preserved).
Inputs: broker export, NFP/FOMC/CPI calendar
Output: raw tick stream + event tags
STAGE 02
Data Validation
Validate tick data integrity: gap detection, timestamp monotonicity, outlier filtering, weekend gap removal, daylight savings adjustment, broker timezone alignment.
Checks: 14 data quality gates
Reject: if > 0.1% ticks fail
STAGE 03
Cost Engine Setup
Load broker cost profile (spread P50/P90/P99, commission RT, swap long/short, slippage distribution). Calibrate against last 30 days of live fills. Verify P50 within ±15% of live.
Calibration: rolling 30-day live comparison
Drift: PSI > 0.25 = recalibrate
STAGE 04
Strategy Execution
Run strategy on tick stream with cost engine applied. Simulate order routing (market/limit/stop/OCO), partial fills, requotes on news, margin calls, latency 100-200 ms. Record every fill with full attribution.
Engine: TickReplayExecutor (C++)
Speed: ~2M ticks/sec
STAGE 05
Metrics Computation
Compute 24 metrics: 6 return (CAGR/total/avg/win-loss), 6 risk (Sharpe/Sortino/Calmar/MDD/Volatility/CVaR), 6 trade (win rate/profit factor/expectancy/avg hold/trades per day/Payette ratio), 6 cost (cost drag/spread cost/commission/swap/slippage/realistic vs idealized).
Output: metrics.json
Benchmarks: vs buy-hold + 1-2-3
STAGE 06
Reporting & Certification
Generate 3 reports (executive / technical / regulatory). Apply failure criteria: 3-band verdict (Pass / Conditional / Reject). Archive to S3 with 7-year retention. Dispatch to engineering.
Verdict: PASS / COND / REJECT
Archive: S3 + signed manifest
Data Validation Checks (Stage 2)
GAP-001
Tick timestamp gaps < 5 seconds during trading hours. Reject if > 1 gap per trading day.
GAP-002
No tick data missing during NFP/FOMC windows (highest-value events).
MONO-001
Timestamps strictly monotonic. No duplicates, no backwards jumps.
OUT-001
Price outliers: |Δtick| > 5×ATR flagged, manually verified.
WKND-001
Weekend gaps (Fri 22:00 → Sun 23:00 UTC) removed, no synthetic fills.
TZ-001
All timestamps normalized to UTC. Broker server TZ offset verified.
Execution Simulator Realism (Stage 4)
LATENCY
Signal-to-broker latency 100-200 ms (P50/P99 from live measurement).
REQUOTE
Market orders during news (spread > 3× baseline) get requoted 70% of time.
PARTIAL
Orders > 5 lots get partial fills (50/30/20 split) at successively worse prices.
MARGIN
Margin calls triggered at ML ≤ 100%. Stop-out at ML ≤ 50%.
SLIPPAGE
Sampled from broker-specific P50/P90/P99 distribution per fill.
SWAP
Triple swap Wednesdays. Compounded daily. Applied at 22:00 GMT.
