Champion vs Challenger — Auto-Retraining Governance
New model must pass Backtest → Walk-Forward → Monte Carlo before promotion · NO live auto-deploy
STAGE 1
Detect Drift
PSI drift detector runs every 6 h. PSI > 0.25 on input features or model confidence triggers retrain.
Trigger: PSI > 0.25
Cadence: 6 h
→
STAGE 2
Train Challenger
Train new model on rolling 90-day window. Parallel to champion — zero production impact.
Train data: 90 days
Time: ~45 min
→
STAGE 3
Backtest (M16)
12-month backtest with realistic costs. Must pass: Sharpe ≥ 2.0, MDD ≤ 5%, cost drag ≤ 35%.
GATE 1 of 3
Verdict: PASS / REJECT
→
STAGE 4
Walk-Forward (M17)
5-7 fold WFA. Must pass: WFE ≥ 0.85, all folds OOS Sharpe ≥ 1.5, OOS MDD ≤ 5%.
GATE 2 of 3
Verdict: PASS / REJECT
→
STAGE 5
Monte Carlo (M18)
10,000 trade permutations. Must pass: survival rate ≥ 95%, P5 Sharpe ≥ 1.0, P5 MDD ≤ 8%.
GATE 3 of 3
Verdict: PASS / REJECT
→
STAGE 6
Manual Promote
If all 3 gates pass, challenger becomes new champion. Manual sign-off required — engineering lead + risk officer + CTO.
Promotion: MANUAL
Auto-deploy: NEVER
⚠ INSTITUTIONAL GUARDRAIL
Auto-retraining NEVER auto-deploys to production
A new model can pass all 3 validation gates and still be rejected at manual review. Reasons for manual rejection include: regime shift not captured by historical data, parameter instability across folds, cost profile drift, or strategic concerns. This is the single most important difference between retail bots and institutional bots. Retail = train → deploy. Institutional = train → validate 3× → manual review → deploy.
RETAIL BOT (typical)
✗ Train on recent data
✗ Auto-deploy to live
✗ No validation gates
✗ No human review
✗ Single model (no champion/challenger)
✗ No rollback path
Result: overfitting → live losses → blown account
TITAN INSTITUTIONAL
✓ Train on rolling 90-day window
✓ Champion stays in production
✓ Challenger runs in parallel (shadow)
✓ 3 validation gates (Backtest/WFA/MC)
✓ Manual review: Eng + Risk + CTO
✓ Rollback: champion restored in < 1 min
Result: validated generalization → live alpha retention
