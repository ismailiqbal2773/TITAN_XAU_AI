STAGE 7
Execution Dispatch
Async order router → broker. Idempotency key. Fill confirmation → audit.
Latency: 21 ms
Throughput: 50 ops/s
Audit: 100%
EVENT BUS
All stages publish/subscribe via NATS JetStream topics: ticks → features → regime → signals → orders → fills → audit. Each stage is independent, horizontally scalable, and replayable from any offset. 3-day retention enables backtesting from production events.
STATE
Stateful services: Risk Engine (position book, exposure), Execution Engine (open orders, fills), Regime Detector (current state). All other stages stateless. State persisted to disk every 100 ms; recovery < 5 seconds.
BACKPRESSURE
If any stage exceeds its latency budget by 2×, downstream stages see "stale signal" flag and the risk engine vetoes execution. Prevents trading on delayed data — critical for institutional safety.
